我的15個網頁設計常用工具懶人包

Sublime text 人氣文字編輯器

我個人最喜歡用的編輯器 Top1
以前學網頁設計都是學Dreamweaver為大宗
對我來說他超肥的，所見即所得有落差，反正就是用不慣(大概只有畫map最好用吧?!)
第一次接觸sublime完全上手也有超豐富外掛可以加
軟體輕量，開啟快速，對電腦效能弱的機種很友好(我的電腦就是很弱怎麼樣XD)
缺點是安裝完打開啥都沒有，要裝的外掛清單都能再寫一篇了XD
但我還是很喜歡他~裝了外掛根本神器!!

寫html+css的推薦使用
寫scss需搭配另一個軟體prepos來編譯

Visual Studio Code 基本完整的開發文字編輯器

我最喜歡的編輯器 Top2
偶爾會看心情選擇用sublime或vs code
跟sublime相比稍肥一咪咪打開也還算快速
一樣可以裝一堆外掛，但是安裝完基本的他都裝好了只須加裝自己需要的就好

寫sass/scss的時候推薦使用，僅須加裝外掛就可以了

codepen 線上文字編輯器

網頁新手練習階段的好夥伴
及時預覽所見即所得，隨寫隨看
有問題要發問，連結一貼方便看
使用搜尋功能召喚以前寫過但忘記的，當成我的私房筆記

Figma UI設計工具

業界最知名UI設計軟體的應該是非sketch莫屬了，但是Mac only
Adobe XD加入戰局，Mac & Microsoft 雙平台可用搶攻市場
一堆討論sketch跟Adobe XD哪個好的討論度非常高
但我個人更喜歡用Figma，他在國外非常火紅但在國內討論度很低
連中文教學都非常少，明明就超好用的阿!!

fakeimg 假圖產生器-文字版

fakeimg是純色背景+圖片尺寸
練習的時候常常只是要把容器填滿，內容不是很重要
修改數字或顏色就可以輕鬆地改圖片尺寸跟背景了，超方便

picsum.photos 假圖產生器-圖片版

在新手村時期我使用fakeimg的頻率超高
連挑圖都懶就用他吧~還可以自動random
一樣可以指定尺寸，還可以指定類別
這樣就不必在尋找圖片上太過費神了

unsplash 免費高質感線上圖庫

超精美高清質感爆棚的線上圖庫
在新手練習階段根本好夥伴
無須下載，只需複製圖片網址
搭配codepen或本機使用都非常方便不怕掉圖(因為是絕對路徑嘛XD)

freepik 免費線上圖庫素材(可商用)

素材圖片的話freepik有很多高質感素材可供下載
免費版每天有限制10次下載次數(加入會員後)
付費版每天最多可下載多達200次

注意免費版使用原則上要註記來源喔!!

font awsome 人氣字體圖標工具套件

最古老的應該就是Windows內建的Webdings字型了吧
以文字取代圖，這麼做的好處是可以大幅減少圖片的大小
跟文字大小一樣控制icon大小，文字可以改的fontawesome都可以改
而且是向量的放大縮小不失真

flaticon 圖標icon(png & svg)

網頁設計中經常會使用icon做裝飾用
flaticon有png & svg格式，有彩色跟單色，有線框跟填滿多種選擇
免登入也可下載，不會畫圖的人也能輕鬆增加網頁豐富度

TinyPNG 線上壓縮圖片工具

幫圖片減肥可以大幅增加網頁下載速度
這個工具可以一口氣把圖片都拖曳進去
他壓縮後”視覺畫質”可接受但檔案大小大幅減少
可選擇單一下載或打包下載

colorable 檢測顏色對比度ratio工具

做無障礙網頁需要知道文字與顏色的對比度是否足夠清晰
等級有分為A AA AAA三種
通常都要符合AA級以上才足夠清晰
不用符合無障礙規範一般網頁也可以用這個工具來看

colorpicker Google顏色選取工具

只要在google搜尋colorpicker就可以成功召喚他啦
很簡單方便的小工具

Google Chrome DevTools 網頁開發人員工具

按右鍵>檢查就會出現開發者工具
可以用來檢查網頁出錯/破版/修bug/測試/換顏色/寫code
網頁開發者必備!!

wappalyzer 網站分析工具

有時候看到一個網站很有趣，想知道這個網站用什麼套件什麼框架
這個瀏覽器的外掛(支援Chrome or Firefox)就派上用場啦
一鍵分析主機環境、內容管理系統、系統平台、網站框架、主機軟體、所使用的分析工具
在學習階段也非常有幫助
不過部分wordpress被修改會偵測不到，刻意把資訊隱藏的也偵測不到

把這些存起來，重灌電腦的時候就不用怕了~哇哈哈哈哈

用Hexo創建免費人氣blog部落格 for Mac

發表於 2020-02-02 | 分類於 Hexo

本篇記錄我第一次使用Hexo安裝的過程，作業系統平台為macOS High Sierra 10.13.6
參考資源：Hexo 官方文件 、 六角學院-使用 Hexo 建構個人化部落格

適合有寫code基礎的族群，使用Markdown做為寫作基礎
其他作業系統如windows或linux也可以用Hexo，安裝步驟可能會跟本篇不太一樣

前置作業

開始之前須準備4個東西

  • Node.js (Node.js 版本需不低於8.10，建議使用 Node.js 10.0 及以上版本)
  • git
  • Xcode (Mac使用者才要裝)
  • Github (註冊帳號密碼，可以當作網路免費空間的概念)

使用spotlight開啟終端機

安裝 Node.js

  1. 使用官方 Node.js 點選LTS版本下載，我的版本是node-v12.14.1
  2. 打開dmg檔執行安裝

安裝 git

  1. 使用官方 git 點選Download下載
  2. 打開dmg檔執行安裝

    跑不過的時候可能會需要下sudo的指令

安裝 Xcode

Hexo文件提及 Mac使用者可能會編譯有問題，所以要再加裝Xcode

  1. 開啟 app store 搜尋 Xcode 下載
  2. 執行安裝

    [!NOTE]
    我在這裡卡關，因為系統跳提示需要 macOS Mojave 10.14.6 才給裝
    所以另外在 Apple Developer尋找降階版本免費安裝
    More Downloads for Apple Developers 輸入帳密登入，尋找最近新的版本下載 我下載的版本是Xcode_9.4.4 (5.27GB很肥)

安裝 Hexo

以上步驟完成後，開啟終端機輸入

寫完文章後，將 Hexo 從本地端推上 Github

  1. 在終端機把服務停掉，輸入指令

  2. 在終端機編譯靜態網站，輸入指令後產生public資料夾

  3. 在Github開一個儲存庫Create a new repository，點選NEW，填入repository name，按Create repository

    注意每個人只有一個主帳號，填入repository name有兩種選擇：

    1. 網址為yourname.github.io，放主帳號根目錄下，請填帳號名稱yourname.github.io
    2. 網址為yourname.github.io/blog，放根目錄下另開新資料夾，repository name請填自訂的資料夾名稱(例如blog)
      差別在於主帳號網址是唯一，另開資料夾網址可以無限多個

  4. 回到終端機，把git建起來

    $ hexo init [folder]建立一個新的網站。
    如果沒有設定 folder 的話，Hexo 會在目前的資料夾建立網站。

  5. 在Github把剛才產生的or push an exsting repository from the command line區塊貼到終端機上

    這段每個人產出的不一樣喔~只是參考，這麼做可以把當下的原始碼推上Github

連接Github把網頁公開發佈，讓所有人看見

  1. 把public資料夾推上Github另一個branch，打開_config.yml檔案
  2. 貼在_config.yml的最下方覆蓋原本的deploy設定

    branch想放主分支請填master

  3. 回到Github複製儲存庫(repo)，點選Clone or Download按鈕顯示儲存庫路徑，貼在repo後
  4. 安裝hexo自動部屬工具
  5. 回到終端機輸入部屬指令，部屬所有內容在Github上

完工!!

重新整理Github可以看到內容已改變，變成public資料夾的內容
branch也多了剛剛新增的gh-pages
在Github的setting找到Github Pages部落格的網址路徑在這兒，點開他就對啦!!

同場加映：Hexo常用終端機指令

運行 $ hexo server
停止 $ ctrl+c
編譯 $ hexo g
發佈 $ hexo d
清除緩存 $ hexo clean
測試 $ hexo s --debug
